Redis Tutorial

What is Redis?

Redis is an open source, in-memory data store used as a database, cache, streaming engine, and message broker. It stands for 'Remote Dictionary Server'. All redis data is stored in the server main memory (RAM) and hence, is exceptionally fast.

redis.io
Advantages of Redis
  • Exceptionally Fast
  • Persistence - While all data lives in memory, changes are asynchronously saved on disk
  • Supports many data types such as lists, sets, hashes etc.
  • Atomicity - All redis operations are atomic, which means if two clients access concurrently, redis server will receive the updated value
  • Multi-utility tool
How to install redis on Windows?
  • From Start menu, open "Turn windows features on and off"
  • Scroll down to find "Windows subsystem for linux", (WSL) and enable it and then restart your pc.
  • Open microsoft store. Search and install Ubuntu
  • Once installed, open Ubuntu from Start menu and set username password for ubuntu
  • Run the following commands:
    • sudo apt-add-repository ppa:redislabs/redis
    • sudo apt-get update
    • sudo apt-get upgrade
    • sudo apt-get install redis-server
How to run redis on Windows?
  • Follow these steps to run redis:
    • Open two Ubuntu Terminals from Start menu
    • In the first terminal, run 'redis-server' without the quotes
    • In the second terminal, run 'redis-cli' without the quotes. In this terminal run the redis operations

Basic Redis Commands

Command Syntax Example Usage
SET SET key_name key_value SET name Debdeep Setting a key-value pair data
GET GET key_name GET name Returns the value of the key. If the key doesn't exists it returns (nil)
DEL DEL key_name DEL name Deletes the key
EXISTS EXISTS key_name EXISTS name Returns 1 if the key exists, else 0
flushall flushall flushall Deletes all the keys and the data
KEYS KEYS pattern KEYS * / KEYS n*e Returns all the keys that matches the pattern, like * will return all the keys and n*e will return all the keys that start with 'n' and ends with 'e'
ttl ttl key_name ttl name Stands for 'time to live'. It returns the expiration time for a key in secs. If -1 is returned, it means it has no expiration time. If -2 is returned, that means the key no longer exists since it is expired
expire expire key_name ttl_in_sec expire name 10 Sets the expiration time for a key. The key will be deleted after ttl secs
setex setex key_name ttl_in_sec key_value setex name 10 debdeep This is a combination of SET and expires

Lists in Redis

Command Syntax Example Usage
lpush lpush list_name list_value1 list_value2 list_value3.... lpush sports Cricket Football Initialize a list in redis. 'l' in 'lpush' stands for left. So this means we are pushing the items in the list from the left
rpush rpush list_name list_value1 list_value2 list_value3.... rpush sports Cricket Football Initialize a list in redis. 'r' in 'rpush' stands for right. So this means we are pushing the items in the list from the right (at the end of the array)
lrange lrange list_name starting_index ending_index lrange sports 0 -1 Returns the items in the list from the left. '0 -1' indexes will return the entire list
lpop lpop list_name count lpop sports 2 Performs a pop operation on 'count' number of items in the list from the left. 'count' is an optional parameter
rpop rpop list_name count rpop sports 2 Performs a pop operation on 'count' number of items in the list from the right. 'count' is an optional parameter

Sets in Redis

Command Syntax Example Usage
SADD SADD set_name value1 value2 value3.... SADD hobbies Singing Dancing "Playing Games" Initialize a set in redis. Does not allow adding duplicate values
SMEMBERS SMEMBERS set_name SMEMBERS hobbies Returns the values in the set
SREM SREM set_name value SREM hobbies Dancing Removes the value from the set

Sorted Sets in Redis

Command Syntax Example Usage
ZADD ZADD set_name score1 value1 score2 value2 score3 value3... ZADD users 2 Deb 1 Deep Initializes the set, sorted on the basic of scores in ascending order
ZRANGE ZRANGE set_name starting_index ending_index withscores ZRANGE users 0 -1 withscores Returns the sorted set. 'withscores' is an optional parameter which returns the set values along with their scores

Hashes in Redis

Command Syntax Example Usage
HSET HSET hash_name key_name key_value HSET person name Debdeep Initialize a hash in redis. It will create a key-value pair with key_name:key_value
HGET HGET hash_name key_name HGET person name Returns the value for the key_name in the hash
HGETALL HGETALL hash_name HGETALL person Returns all the keys and values separately in the hash
HDEL HDEL hash_name key_name HSET person name Deletes the key-value pair
HEXISTS HEXISTS hash_name key_name HEXISTS person name Returns 1 if the key exists in the hash, else 0

Pub/Sub in Redis

PubSub is the messaging system in Redis where Sender is the Publisher and Receiver is the Subscriber. A channel can have multiple subscribers. To subscribe (or listen) to a channel, we use SUBSCRIBE channel_name. To send a message to a channel, we use PUBLISH channel_name message_content. PSUBSCRIBE pattern such as PSUBSCRIBE a*b will subscribe to all the channels that matches the pattern. To stop listening to a channel, use UNSUBSCRIBE channel_name. To unsubscribe to channels based on a pattern, use PUNSUBSCRIBE pattern to unsubscribe from all channels based on a pattern.

Use Cases

  • Chat applications: Users can subscribe to a chatroom where a single user can publish a message and that will be available to all the users subscribed to the channel.
  • Message Queue: All those who want jobs to be done can publish the messages in a queue, and the Processing Unit can subscribe to the channel, listen to the requests one by one process then and return the results.
